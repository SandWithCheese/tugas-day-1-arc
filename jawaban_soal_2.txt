Git rebase adalah perintah Git yang memungkinkan untuk mengubah urutan, mengedit, atau menggabungkan commit dari satu branch ke branch lainnya. Dengan git rebase, kita bisa memodifikasi sejarah repositori sesuai keinginan. Git rebase biasanya digunakan sebagai alternatif dari git merge. Git merge menggabungkan dua cabang dengan membuat commit baru yang berisi catatan penggabungan. Git rebase menggabungkan dua cabang dengan menerapkan commit dari satu cabang di atas commit dari cabang lainnya. Ini membuat sejarah repositori lebih bersih dan mudah dibaca. Namun, git rebase juga memiliki beberapa kekurangan, seperti sulit diperbaiki jika terjadi konflik saat rebase, dan berpotensi mengubah commit yang sudah ada di repositori publik.
