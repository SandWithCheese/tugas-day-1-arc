Merge conflict adalah sebuah situasi di mana terdapat pertentangan antara dua sumber kode yang berbeda saat proses merge dengan Git. Ini bisa terjadi karena ada perubahan yang saling bertentangan pada file atau komponen yang sama. Untuk menanggulangi merge conflict, dapat menggunakan VS Code untuk membandingkan dan memilih perubahan yang diinginkan. Buka file yang bermasalah dengan VS Code dan cari tanda <<<<<<<, =======, dan >>>>>>> yang menunjukkan konflik. Pilih perubahan yang diinginkan, hapus tanda-tanda tersebut, lalu simpan file. Setelah itu, gunakan perintah git add dan git commit untuk menyelesaikan proses merge.
